# Security Policy

## Reporting a vulnerability

If you believe you have found a security vulnerability, please **do not** open a public GitHub Issue.

Preferred reporting methods:

1. **GitHub Security Advisories** (Private Vulnerability Reporting) for this repository
2. Email: **security@enterpriseglue.ai**

Please include as much information as possible:

- Affected component(s) (backend/frontend)
- Steps to reproduce
- Impact and severity assessment (if known)
- Any proof-of-concept code or logs

## Response targets

- Acknowledgement: best-effort, typically within **72 hours**
- Fix and disclosure: coordinated disclosure; timelines depend on severity and complexity

## Supported versions

Security fixes are provided for the **latest released version**.

## Disclosure

Please allow time for investigation and remediation before public disclosure.

## Out of scope

The following are generally out of scope for vulnerability reporting:

- Social engineering
- Denial of service requiring unrealistic traffic volumes
- Issues that only affect self-hosted deployments due to misconfiguration
